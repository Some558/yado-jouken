## プロンプト本文(ここから下をコピー)

温泉宿のデータベースサイト「ヤドシボリ」の全ページデザインを作ってください。

### サイトの役割

「泊まりたい条件が先にあって、エリアはどこでもいい」人のための温泉宿データベースです。例:「ペットと泊まれる温泉地はどこが充実している?」「貸切風呂の相場は箱根と草津でどう違う?」。楽天トラベルの公開データを毎日自動集計し、条件×温泉地の掛け合わせ約80ページで「該当件数・料金相場・レビュー平均」を比較できます。

- 訪問者: 条件が決まっている旅行者(ペット連れ・子連れ・サウナ好き・カップル等)。検索エンジンから条件×エリアのページに直接着地します
- 最優先の導線: ①各宿の「プランを見る」ボタン(楽天トラベルへの送客・収益源) ②条件⇔エリアの回遊(早見表・ハブページ)
- サイト名: **ヤドシボリ**(=条件で宿を絞る)。タグライン「泊まりたい条件から探す、温泉地の宿データベース」。ロゴはテキストロゴで良いですが「絞る」を感じる遊び(絞り柄・湯けむり等のワンポイント)は歓迎

### デザインの方向性

- **トラベル雑誌のような写真リッチで華やかな誌面**。施設写真(横長・楽天トラベル提供のCDN画像)を宿一覧カードの主役にする。モックではプレースホルダ写真で構いません
- 温泉旅の情緒(湯けむり・夕暮れの温泉街のような温かい色気)と、データサイトとしての信頼感(数字が読みやすい・整然)の両立
- 避けたい方向: 無機質な企業比較サイト風 / 白いカードに影を付けて並べただけの単調なレイアウト / 紺+ベージュの地味な和モダン(現行デザインからの脱却が目的)
- 数字は主役級に扱う: 「該当14件」「最安¥4,800〜」のようなスタットは大きく・楽しく

### 画面構成(5テンプレート・各PC+スマホ)

1. **トップ**: ヒーロー(サイト名+タグライン+役割説明)→ 条件×エリアの早見表(8条件×10エリア・セル=該当施設数のリンク)→ 条件から探すカード8枚 → エリアから探すカード10枚
2. **条件×エリア詳細**(本体・最重要): パンくず → H1「{エリア}の{条件}の宿」→ リード文(該当件数・割合・最安・中央値のデータ差し込み文)→ スタット4枚(該当施設数/最安の最低料金(大人1名あたり)/最低料金の中央値/掲載施設のレビュー平均)→ 条件の解説ボックス(短文)→ **宿一覧(最大20件・写真+施設名+アクセス+最低料金+レビュー★+「プランを見る」ボタン)** → エリア解説 → データの根拠ボックス(「このページのデータについて」: 取得日・判定方法・分母の箇条書き)→ 回遊リンク2枚
3. **条件ハブ**: H1「{条件}の宿をエリアで比較」→ エリア横断の比較表(エリア/該当/該当率/最安/料金中央値/レビュー平均)→ 条件解説
4. **エリアハブ**: H1「{エリア}の宿を条件から探す」→ エリア解説 → 条件リンクカード8枚
5. **静的ページ**(データの取得方法・運営者情報など): 本文が読みやすい素直な1カラム。凝らなくてよい

### 確定している文言(変更しないでください)

- 全ページヘッダー直下に常時表示: 「本サイトはアフィリエイト広告(楽天アフィリエイト)を利用しています。」(小さくてよいが省略・折りたたみ不可)
- フッターに固定: 「Supported by Rakuten Developers」(テキストリンク)+ナビ「データの取得方法/運営者情報/免責事項/プライバシーポリシー」+「掲載データは楽天トラベルの公開情報をAPI経由で取得し、毎日自動更新しています。料金・空室状況は変動するため、最新情報は必ずリンク先でご確認ください。」
- スタットのラベル4種・表ヘッダー(施設名/最低料金/レビュー/エリア/該当/該当率/料金中央値/レビュー平均)・ボタン文言「プランを見る」・ボックス見出し「このページのデータについて」
- 各ページに「データ取得日: YYYY-MM-DD」を表示する場所を必ず設けてください(信頼性の要)

### 宿1件が持つデータ(一覧カードの設計材料)

施設写真(横長)/施設名/かな/アクセス(例「箱根湯本駅から徒歩7分」)/最低料金(例 ¥10,182〜)/レビュー平均(例 ★4.25)とレビュー件数(例 1,234件)/ひとこと紹介文(施設側の宣伝文・1行)

### 技術制約

- 静的サイト(Astro)にプレーンCSSで移植します。**デザイントークン(色HEX・フォント名・角丸・余白スケール・シャドウ)を README に一覧化**してください
- フォントは Google Fonts の無料日本語フォントまたはシステムフォント
- JavaScript 必須の動き(カルーセル・無限スクロール等)は使わない。CSSだけで成立するレイアウトに
- レスポンシブ必須(スマホ=1カラム・比較表はコンテナ内横スクロール許容)。ダークモード不要
- 写真は外部CDN参照のため、読み込み失敗時も崩れないカード設計(背景色フォールバック)を考慮

### 納品形式

hi-fi モックアップ(5テンプレート×PC/スマホ)+デザイントークン一覧+レイアウト仕様の README。
